How to Create a Custom Redirect with Cloudflare Pages

In this article, we will guide you through the process of creating a custom redirect for your website using Cloudflare Pages. Cloudflare Pages is a powerful platform that allows you to deploy and host your static websites easily. By following these steps, you'll be able to set up a redirect to another URL of your choice, giving you full control over your website's redirection.

Step 1: Sign up for Cloudflare Pages

Visit the Cloudflare website ( pages.cloudflare.com) and sign up for an account if you haven't already. Once signed in, click on "Get Started" to create a new project.

Step 2: Set up your Cloudflare Pages project

Choose your preferred Git provider and connect your repository that contains the website you want to redirect from. Configure your build settings based on your website's setup (e.g., static site generator, build command, and output directory). Review and confirm your project settings, then click on "Begin Setup."

Step 3: Make the redirect

In your Cloudflare Pages Git repo, make a new file called _redirects. Edit the file to your liking with this example: /source https://domain.tld/. For multiple domains, just make a new line and do the same thing.

Step 4: Deploy your changes

Cloudflare Pages will automatically detect the changes and trigger a new build and deployment process. Wait for the deployment process to complete, and your website with the custom redirect will be live.

Step 5: Testing the redirect

Open a web browser and visit your website's URL. The custom redirect should now be in effect, redirecting visitors to the specified URL.
